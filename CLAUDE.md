# Indaspro - Home Services Platform

## Project Overview
Indaspro is a hyperlocal home services marketplace for small towns in West Bengal, India.
It is an Urban Company clone optimized for a zero-cost pilot.
Single PWA for Customer + Provider + Admin — role-based, one codebase.

- **App Name:** Indaspro
- **Firebase Project ID:** indaspro-80195
- **Vercel URL:** indaspro.vercel.app
- **Target Market:** Bankura / Bishnupur / Durgapur, West Bengal
- **Languages:** Bengali (primary), Hindi, English
- **Tagline:** "আপনার ঘরের সেবা, এক ক্লিকে" (Your home service, one click away)

---

## Tech Stack

- **Framework:** Next.js 14 (App Router, TypeScript, strict mode)
- **Styling:** Tailwind CSS + shadcn/ui components
- **Icons:** lucide-react + emoji for service categories
- **Backend:** Firebase Spark (free plan, NO credit card)
  - Firestore (NoSQL database)
  - Firebase Auth (Phone OTP — 10K free/month)
  - NO Cloud Storage (pilot phase — no photo uploads)
- **Hosting:** Vercel (free tier)
- **PWA:** next-pwa for installable mobile app
- **Forms:** react-hook-form + zod validation
- **Dates:** date-fns
- **Toasts:** react-hot-toast
- **QR Code:** qrcode.react (for UPI payment QR)
- **i18n:** next-intl (Bengali/Hindi/English)

---

## Key Architecture Decisions

1. **Single app with role switching** — user registers once, can be Customer, Provider, or Both. Role switch via profile menu.
2. **No Google Maps** — text-based addresses with landmark field ("রামকৃষ্ণ মন্দিরের পাশে"). Small town, everyone knows landmarks.
3. **No photo uploads** — no Cloud Storage needed, stays on Spark free plan. Avatars show name initials in colored circle. Provider verification done in person. Payment proof via UTR number text input only.
4. **Firebase Phone Auth** — built-in OTP via SMS, 10K free/month. Returning users use 4-digit PIN (stored as hash in Firestore) to avoid OTP cost.
5. **UPI QR + Cash payments** — no Razorpay, no payment gateway. Static UPI QR image generated client-side via qrcode.react. Customer enters UTR number after paying. Admin verifies against bank statement.
6. **Bengali-first UI** — all user-facing strings in i18n JSON files. UI labels in Bengali with English fallback.
7. **Mobile-first PWA** — designed for Android phones on 2G/3G. Lightweight, installable, works offline for viewing past bookings.
8. **Denormalized Firestore** — provider name, customer name, address embedded in booking docs to minimize reads (free tier quota).
9. **Realtime booking tracking** — Firestore onSnapshot listener on individual booking document for live status updates.

## Firestore Security Rules

```javascript
// firestore.rules
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    match /users/{userId} {
      allow read: if request.auth != null;
      allow write: if request.auth.uid == userId;

      match /addresses/{addressId} {
        allow read, write: if request.auth.uid == userId;
      }
      match /notifications/{notifId} {
        allow read: if request.auth.uid == userId;
        allow write: if request.auth != null;
      }
    }

    match /providerProfiles/{userId} {
      allow read: if request.auth != null;
      allow create: if request.auth.uid == userId;
      allow update: if request.auth.uid == userId || isAdmin();
    }

    match /serviceCategories/{doc} {
      allow read: if true;
      allow write: if isAdmin();
    }

    match /services/{doc} {
      allow read: if true;
      allow write: if isAdmin();
    }

    match /bookings/{bookingId} {
      allow read: if request.auth != null
                  && (resource.data.customerId == request.auth.uid
                      || resource.data.providerId == request.auth.uid
                      || isAdmin());
      allow create: if request.auth != null;
      allow update: if request.auth != null
                    && (resource.data.customerId == request.auth.uid
                        || resource.data.providerId == request.auth.uid
                        || isAdmin());

      match /extras/{extraId} {
        allow read, write: if request.auth != null;
      }
    }

    match /reviews/{reviewId} {
      allow read: if true;
      allow create: if request.auth != null
                    && request.resource.data.customerId == request.auth.uid;
    }

    match /coupons/{doc} {
      allow read: if true;
      allow write: if isAdmin();
    }

    match /settlements/{doc} {
      allow read: if request.auth != null
                  && (resource.data.providerId == request.auth.uid || isAdmin());
      allow write: if isAdmin();
    }

    match /config/{doc} {
      allow read: if true;
      allow write: if isAdmin();
    }

    function isAdmin() {
      return request.auth != null
             && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.roles.hasAny(['admin']);
    }
  }
}
```

## Coding Standards

1. **TypeScript strict mode** — no `any` types
2. **Functional components** with hooks only
3. **shadcn/ui** for all base components (Button, Input, Card, Dialog, Tabs, Sheet, Badge, etc.)
4. **Tailwind** for all styling — no CSS modules, no styled-components
5. **Mobile-first** — design for 360px width, scale up
6. **Firebase calls** only through src/lib/ helper functions, never directly in components
7. **Error handling** — try/catch all Firebase operations, show toast on error
8. **Loading states** — use LoadingSpinner for all async operations
9. **Empty states** — use EmptyState component when lists are empty
10. **Form validation** — zod schemas for all forms, react-hook-form for handling
11. **Bengali-first** — all UI strings from i18n, Bengali is default
12. **Accessibility** — proper aria labels, semantic HTML
13. **InitialsAvatar** — generate from first letter of name, consistent color based on name hash
14. **Price formatting** — always "₹XXX" format using formatPrice utility
15. **Date formatting** — use date-fns with Bengali locale where possible
16. **Booking number format** — IND-YYYYMMDD-XXX (auto-increment)
17. **OTP** — 4-digit random number, stored in booking doc
18. **PIN** — 4-digit, hashed with bcryptjs before storing
